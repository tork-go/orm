# Installation

## Requirements

- Go 1.25 or later. Tork's own `go.mod` requires it, mainly because `pgx/v5`,
  the PostgreSQL driver it's built on, raised its own minimum Go version in
  recent releases.
- A PostgreSQL database. SQLite and MySQL support are planned but not built yet.

## Add the module

```sh
go get github.com/tork-go/orm
```

This gives you the `orm` package (model definitions) and the `schema` package
(used internally by migrations). If you're going to run migrations against
PostgreSQL, which you almost certainly are, also install the driver:

```sh
go get github.com/tork-go/orm/driver/postgres
```

That pulls in `github.com/jackc/pgx/v5` as a transitive dependency. Nothing else
in Tork depends on pgx: the driver package is the only place it's imported, so
adding SQLite or MySQL support later never touches your model definitions.

## A local PostgreSQL instance

For development, the simplest option is Docker:

```sh
docker run -d --name tork-dev \
  -e POSTGRES_USER=tork \
  -e POSTGRES_PASSWORD=tork \
  -e POSTGRES_DB=tork_dev \
  -p 5432:5432 \
  postgres:16-alpine
```

Or with a `docker-compose.yml`:

```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: tork
      POSTGRES_PASSWORD: tork
      POSTGRES_DB: tork_dev
    ports:
      - "5432:5432"
```

Then `docker compose up -d`. Either way, your connection string for the rest of
these docs is:

```
postgres://tork:tork@localhost:5432/tork_dev?sslmode=disable
```
