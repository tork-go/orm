# Connecting

```go
import (
	"github.com/tork-go/orm"
	_ "github.com/tork-go/orm/driver/postgres"
)

db, err := orm.Connect(ctx, "postgres://tork:tork@localhost:5432/app")
if err != nil {
	return err
}
defer db.Close(ctx)
```

That `db` is what every query takes:

```go
users, err := Users.With(db).Where(Users.Age.GreaterOrEqual(18)).All(ctx)
```

## Why the blank import

Go cannot load a package by name at run time, so something has to link the
driver in. Tork's own `orm` package deliberately does not: importing every
driver would put PostgreSQL's client, SQLite's, and MySQL's into your binary
whichever one you actually use. Instead each driver registers itself when it is
linked, and the blank import is what links it. This is the same arrangement
`database/sql` uses.

You can check that it worked:

```sh
go list -deps github.com/tork-go/orm | grep -i pgx
```

Nothing. The `orm` package has no driver dependency at all — your `main` has.

Forget the import and the error says exactly what to add:

```
orm: no driver named "postgres", and no drivers are registered at all;
    add the blank import that links one in, as
	_ "github.com/tork-go/orm/driver/postgres"
```

## From the parts

A connection string assembled by hand is a common source of quoting mistakes,
usually a password with punctuation in it. Give the parts instead and Tork
escapes them:

```go
db, err := orm.Open(ctx, orm.Config{
	Driver:   "postgres",
	Host:     "db.internal",
	Port:     5432,
	User:     "tork",
	Password: os.Getenv("DB_PASSWORD"),   // p@ss/w:rd? is fine
	Database: "app",
	Options:  map[string]string{"sslmode": "require"},
})
```

`Options` are the driver-specific settings that would otherwise be query
parameters on the URL.

## Pooling

`Connect` and `Open` both return a **pool**, not a single connection. A
connection serves one statement at a time, so a server handling requests
concurrently would otherwise serialise on it.

```go
db, err := orm.Open(ctx, orm.Config{
	URL:             os.Getenv("DATABASE_URL"),
	MaxConns:        20,
	MinConns:        2,
	MaxConnLifetime: time.Hour,
	MaxConnIdleTime: 5 * time.Minute,
})
```

Every pool setting left at zero keeps the driver's own default, so you only
name the ones you actually want to change. `MaxConns: 1` gives you a single
connection if that is genuinely what you want.

`Connect` verifies the database is reachable before returning, so a wrong host
or password is an error here rather than a surprise at your first query.

## One handle, for the life of the program

Open it once at startup and pass it around. It is safe for concurrent use, and
closing it closes every connection in the pool:

```go
func main() {
	ctx := context.Background()

	db, err := orm.Connect(ctx, os.Getenv("DATABASE_URL"))
	if err != nil {
		log.Fatal(err)
	}
	defer db.Close(ctx)

	if err := migrate.Apply(ctx, os.Getenv("DATABASE_URL"), "migrations"); err != nil {
		log.Fatal(err)
	}

	// ... start serving, passing db to whatever needs it
}
```

## Migrations connect the same way

Nothing takes a dialect. The connection string says which database it is:

```go
err := migrate.Apply(ctx, dsn, "migrations")
os.Exit(cli.Run(dsn, "migrations", models.Users, models.Posts))
```

Both need the same blank import, and report the same error without it. See
[Building a CLI](/docs/3.migrations/6.building-a-cli).

## Supported databases

| Scheme | Driver | Status |
|---|---|---|
| `postgres://`, `postgresql://` | `github.com/tork-go/orm/driver/postgres` | built |
| `sqlite://` | — | planned |
| `mysql://` | — | planned |

A driver registers whichever schemes it answers to, so both Postgres spellings
reach the same one.
