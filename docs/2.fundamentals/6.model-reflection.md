# Model Reflection

The migrations system needs to look at an arbitrary model, `User`, `Post`,
whatever you defined, and enumerate its columns without knowing each field's
concrete `T` ahead of time. This page covers how, since it's also useful if you
ever want to inspect a model programmatically yourself.

## `Columns` and `ForeignKeys`

```go
func Columns(m Model) []ColumnMeta
func ForeignKeys(m Model) []ForeignKeyMeta
```

```go
for _, col := range orm.Columns(models.User) {
	fmt.Println(col.Name(), col.GoType(), col.IsNullable())
}
// id int false
// username string false
// email *string true

for _, fk := range orm.ForeignKeys(models.Post) {
	fmt.Println(fk.Name(), "->", fk.ReferencedTable(), fk.ReferencedColumn())
}
// author_id -> users id
```

`Columns` returns every field that's a column, including foreign key fields
(`ForeignKey[T]` embeds `Column[T]`, so it's a column too). `ForeignKeys`
returns only the foreign key fields. Both walk the model struct's exported
fields in declaration order.

## Why this works without knowing `T`

None of `Column[T]`'s or `ForeignKey[T]`'s methods mention `T` in a parameter or
return type, `T` only appears in the receiver's type parameter. That means
`*Column[int]`, `*Column[string]`, and `*Column[*string]` all have exactly the
same method set, and so do all instantiations of `*ForeignKey[T]`. Two small
interfaces capture that shared method set:

```go
type ColumnMeta interface {
	Name() string
	IsPrimaryKey() bool
	IsUnique() bool
	IsNotNull() bool
	MaxLength() (int, bool)
	GoType() reflect.Type
	IsNullable() bool
	IsIndexed() bool
	ServerDefaultExpr() (string, bool)
	IsClientGenerated() bool
}

type ForeignKeyMeta interface {
	ColumnMeta
	ReferencedTable() string
	ReferencedColumn() string
}
```

`Columns`/`ForeignKeys` reflect over a model's fields and keep the ones that
satisfy `ColumnMeta`/`ForeignKeyMeta`. Fields that don't, the embedded `Table`
(only has `TableName()`), and the relationship marker fields (`HasMany`,
`BelongsTo`, `HasOne`, `ManyToMany`, no methods at all), are excluded
automatically, by the type assertion simply failing for them. A nil
`*Column[T]` field is skipped rather than causing a panic.

Not every `Column[T]` method makes it onto `ColumnMeta`: `Generator() (func()
T, bool)` (see [Columns](/docs/2.fundamentals/2.columns)) mentions `T` in its
return type, so it can't join a shared, `T`-free interface. It stays reachable
only on the concrete `*Column[T]`, which is fine, nothing needs it generically
yet.

## `Indexer`: model-level, not field-level

`Columns` and `ForeignKeys` reflect over a model's *fields*. Compound indexes
and unique constraints aren't a field, they're declared with a method on the
model itself, `Indexes() []IndexDef` (see
[Constraints](/docs/2.fundamentals/4.constraints)). `schema.ExtractSchema`
checks for this with a plain type assertion, not reflection, since a Go
method either exists on a type or it doesn't:

```go
if indexer, ok := m.(orm.Indexer); ok {
    for _, def := range indexer.Indexes() {
        // ...
    }
}
```

Implementing `Indexes()` is optional. Most models don't, and `ok` is simply
`false` for them.

## Where this is used

`schema.ExtractSchema`, which `makemigrations` calls to see your models' desired
shape, is built entirely on `Columns`, `ForeignKeys`, and `Indexer`. See
[Generating Migrations](/docs/3.migrations/3.generating-migrations).
