# Model Reflection

The migrations system needs to look at an arbitrary model, `User`, `Post`,
whatever you defined, and enumerate its columns without knowing each field's
concrete `T` ahead of time. This page covers how, since it's also useful if you
ever want to inspect a model programmatically yourself.

## `Columns` and `ForeignKeys`

```go
func Columns(m Model) []ColumnMeta
func ForeignKeys(m Model) []ForeignKeyMeta
```

```go
for _, col := range orm.Columns(models.Users) {
	fmt.Println(col.Name(), col.GoType(), col.IsNullable())
}
// id int false
// username string false
// email *string true

for _, fk := range orm.ForeignKeys(models.Posts) {
	fmt.Println(fk.Name(), "->", fk.ReferencedTable(), fk.ReferencedColumn())
}
// author_id -> users id
```

`Columns` returns every field that's a column, including foreign key fields
(a foreign key is an ordinary column carrying a reference, so it appears in
both). `ForeignKeys`
returns only the foreign key fields. Both walk the model struct's exported
fields in declaration order.

## Why this works without knowing `T`

None of the methods below mention `T` in a parameter or
return type, `T` only appears in the receiver's type parameter. That means
`*Column[int]`, `*Column[string]`, and `*Column[*string]` all have exactly the
same method set, and so do the typed column types (`*StringColumn`,
`*IntColumn`, and the rest, which republish it). Two small
interfaces capture that shared method set:

```go
type ColumnMeta interface {
	Name() string
	OwnerTable() string
	IsPrimaryKey() bool
	IsUnique() bool
	HasNotNull() bool
	MaxLength() (int, bool)
	GoType() reflect.Type
	IsNullable() bool
	IsIndexed() bool
	ServerDefaultExpr() (string, bool)
	IsClientGenerated() bool
	NumericPrecisionScale() (precision, scale int, ok bool)
	IsJSON() bool
	IsJSONB() bool
	IsSerialized() bool
	EnumSpec() (typeName string, values []string, ok bool)
}

type ForeignKeyMeta interface {
	ColumnMeta
	ReferencedTable() string
	ReferencedColumn() string
	OnDeleteAction() ForeignKeyAction
	OnUpdateAction() ForeignKeyAction
}
```

`Columns`/`ForeignKeys` reflect over a model's fields and keep the ones that
satisfy `ColumnMeta`/`ForeignKeyMeta`. Fields that don't, the embedded `Table`
(only has `TableName()`), and the relationship marker fields (`HasMany`,
`BelongsTo`, `HasOne`, `ManyToMany`, no methods at all), are excluded
automatically, by the type assertion simply failing for them. A nil
`*Column[T]` field is skipped rather than causing a panic.

Not every `Column[T]` method makes it onto `ColumnMeta`: `Generator() (func()
T, bool)` and `Serializer() (func(T) ([]byte, error), func([]byte) (T,
error), bool)` (see [Columns](/docs/2.fundamentals/2.columns)) both mention
`T` in their return type, so neither can join a shared, `T`-free interface.
They stay reachable only on the concrete `*Column[T]`, which is fine,
nothing needs them generically yet.

## `Indexer` and `Checker`: model-level, not field-level

`Columns` and `ForeignKeys` reflect over a model's *fields*. Compound indexes
and unique constraints, and CHECK constraints, aren't a field, they're
declared with a method on the model itself, `Indexes() []IndexDef` and
`Checks() []CheckDef` (see [Constraints](/docs/2.fundamentals/4.constraints)).
`schema.ExtractSchema` checks for both with a plain type assertion, not
reflection, since a Go method either exists on a type or it doesn't:

```go
if indexer, ok := m.(orm.Indexer); ok {
    for _, def := range indexer.Indexes() {
        // ...
    }
}
if checker, ok := m.(orm.Checker); ok {
    for _, def := range checker.Checks() {
        // ...
    }
}
```

Implementing `Indexes()`/`Checks()` is optional. Most models don't, and `ok`
is simply `false` for them.

## Where this is used

`schema.ExtractSchema`, which `makemigrations` calls to see your models' desired
shape, is built entirely on `Columns`, `ForeignKeys`, and `Indexer`. See
[Generating Migrations](/docs/3.migrations/3.generating-migrations).

## `ValueCodec`, the other half

`ColumnMeta` deliberately mentions no `T`, which is what makes every
`Column[T]` share one method set. The cost is that `Generator` and
`Serializer`, which both name `T`, can't be part of it. `ValueCodec` says the
same things without naming `T`:

```go
type ValueCodec interface {
	GenerateAny() (any, bool)
	MarshalAny(v any) ([]byte, error)
	UnmarshalAny(b []byte) (any, error)
}
```

Every column satisfies both. It's kept off `ColumnMeta` rather than folded in
so that interface stays a pure description of a column, the same way `Indexer`
and `Checker` are optional interfaces on a model rather than methods on
`Model`. See [Reading Rows](/docs/2.fundamentals/7.reading-rows).

## Relationship markers are not columns

`HasMany[E]`, `HasOne[E]`, `BelongsTo[E]` and `ManyToMany[E]` don't satisfy
`ColumnMeta`, so the walk skips them and they never reach a migration.

They aren't empty types any more, though. `DefineTable` attaches each one to
its table so it can later work out which foreign key joins the two, which is
covered in [Relationships](/docs/2.fundamentals/5.relationships). Binding
writes to the marker the model holds rather than to a copy, so a model
carrying relationships has to be returned as a pointer from the function
passed to `DefineTable`.
