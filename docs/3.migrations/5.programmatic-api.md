# Programmatic API

The CLI (see [Applying Migrations](/docs/3.migrations/4.applying-migrations))
is a thin wrapper over functions in the `migrate` package. You can call them
directly, and for the common case of "bring the database up to date," there's
a single function built for exactly that.

## `Apply`, for your application's startup

```go
func Apply(ctx context.Context, dsn, dir string) error
```

```go
err := migrate.Apply(ctx, dsn, "migrations")
```

This connects, loads every migration file in `dir`, and applies whatever's
pending, up to head. It's meant to be called once, every time your application
starts, the same role as SQLAlchemy's `metadata.create_all(engine)` or
Drizzle's `migrate(db, {...})`. If there's nothing pending, it's a no-op, so
it's safe to call unconditionally rather than guarding it behind an
environment check.

## `Up`, `Down`, and `History` directly

For anything more specific than "apply everything," these are what `Apply` and
the CLI both call underneath:

```go
func Up(ctx context.Context, dialect driver.Dialect, conn driver.Conn, migrations []Migration, target Target) error
func Down(ctx context.Context, dialect driver.Dialect, conn driver.Conn, migrations []Migration, target Target) error
func History(ctx context.Context, dialect driver.Dialect, conn driver.Conn, migrations []Migration) ([]Entry, error)
```

You supply the connection and the loaded migrations yourself:

```go
dialect, err := driver.For(dsn)
if err != nil {
    return err
}

conn, err := dialect.Connect(ctx, dsn)
if err != nil {
    return err
}
defer conn.Close(ctx)

if err := dialect.EnsureHistoryTable(ctx, conn); err != nil {
    return err
}

migrations, err := migrate.LoadAll("migrations")
if err != nil {
    return err
}

if err := migrate.Up(ctx, dialect, conn, migrations, migrate.HeadTarget()); err != nil {
    return err
}

entries, err := migrate.History(ctx, dialect, conn, migrations)
if err != nil {
    return err
}
for _, e := range entries {
    fmt.Println(e.Migration.Revision, e.Applied)
}
```

## Building a `Target` in code

The CLI parses `"head"`, `"base"`, `"+2"`, and so on with `ParseTarget`. From
Go, you can either call `ParseTarget` on a string, or construct one directly:

```go
migrate.HeadTarget()
migrate.BaseTarget()
migrate.StepsTarget(2)
migrate.RevisionTarget("2773adc3c811")

target, err := migrate.ParseTarget("-1")
```

`Up` rejects a `BaseTarget`, `Down` rejects a `HeadTarget`, the same rule the
CLI enforces, whichever way you construct the target.
