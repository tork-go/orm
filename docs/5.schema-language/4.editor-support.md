# Editor support

```
go install github.com/tork-go/orm/cmd/tork-lsp@latest
```

`tork-lsp` is a Language Server Protocol server speaking JSON-RPC over
its standard input and output. It takes no arguments; your editor
starts it and does the rest.

It answers from the same lexer, parser, analyzer, and formatter the
generator uses, so what the editor tells you and what
`go run ./cmd/generate` tells you are the same findings in the same
words. There is no second implementation to drift.

## What it does

- **Diagnostics** as you type, against the buffer rather than the file
  on disk, so nothing waits for a save. Findings are published for
  every file in the schema directory, because editing one file
  routinely fixes or breaks another.
- **Completion**, aware of where the cursor is: declaration keywords at
  the top level, types in type position including models and enums
  from other files, attributes after `@`, native types after `@db.`,
  block attributes after `@@`, field names inside `[...]`, argument
  names inside `@relation(...)`, and referential actions after
  `onDelete:`. Every suggestion carries a line of documentation.
- **Hover**, showing a field's column, its Go type, and its
  constraints; a relation's kind and the key behind it; a model's
  table and primary key; an enum's database type and values; and what
  each attribute does.
- **Go to definition**, across files. On a type it jumps to the
  declaration; on a field named in `fields:` or an index it jumps to
  the field; on a relation field it jumps to the other side. On a
  model's own name it lists the places that refer to it, which is the
  question actually worth asking there.
- **Formatting**, the same canonical form `go run ./cmd/generate fmt`
  writes.

All of it keeps working while a file is mid edit. The parser recovers
from errors rather than stopping at the first one, so a missing brace
costs you a squiggle, not every other feature in the file.

## VS Code

The extension lives in `editors/vscode` in this repository. It adds
syntax highlighting and starts `tork-lsp` for everything else.

```
cd editors/vscode
npm install
npm run compile
```

Press F5 for an Extension Development Host, or `npx vsce package` for
an installable `.vsix`.

It looks for the server on `PATH`, then `$GOBIN`, then `$GOPATH/bin`,
then `~/go/bin`, asking the Go toolchain itself for the last two
rather than guessing. Set `tork.lsp.path` if yours lives elsewhere.

## Other editors

Any LSP client works. The server needs no initialization options; point
your client at the `tork-lsp` executable for the `tork` language and
attach it to `.tork` files. For Neovim with `nvim-lspconfig`:

```lua
vim.filetype.add({ extension = { tork = "tork" } })
require("lspconfig.configs").tork = {
  default_config = {
    cmd = { "tork-lsp" },
    filetypes = { "tork" },
    root_dir = require("lspconfig.util").root_pattern("go.mod", ".git"),
  },
}
require("lspconfig").tork.setup({})
```
